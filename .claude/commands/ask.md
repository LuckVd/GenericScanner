# `/ask` Command

> 只读问答模式 — 分析、解释、不修改

---

## 核心原则

### 绝对只读

- ❌ 不修改任何文件
- ❌ 不创建文件
- ❌ 不删除文件
- ❌ 不生成补丁
- ❌ 不执行写入操作

### 允许操作

- ✅ 读取项目所有文件
- ✅ 分析代码结构
- ✅ 搜索代码
- ✅ 输出解释、建议、重构思路（仅文字）

### 联网策略

- ⚠️ 联网前必须征得用户同意
- ⚠️ 用户可拒绝联网请求

---

## 使用方式

```
/ask <question>
```

**示例：**
```
/ask 这个项目的认证流程是怎样的？
/ask 为什么这个函数会报错？
/ask 帮我分析这个模块的架构
```

---

## 执行流程

1. 声明进入 `READ-ONLY MODE`
2. 解析用户问题
3. 读取相关文件
4. 如需联网 → 请求授权
5. 输出纯文本回答
6. 退出命令

---

## 能力矩阵

| 能力 | 允许 |
|------|------|
| 读取文件 | ✅ |
| 搜索代码 | ✅ |
| 代码解释 | ✅ |
| 结构分析 | ✅ |
| 架构建议（文字） | ✅ |
| 生成修改补丁 | ❌ |
| 自动修复代码 | ❌ |
| 创建/删除文件 | ❌ |

---

## 拒绝示例

```
/ask 帮我修复这个 bug

→ 该命令运行于只读模式，不能修改代码。
请使用普通对话模式进行修改操作。
```
